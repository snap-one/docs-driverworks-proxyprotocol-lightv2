## hide\_proxy\_conditionals\_loops

Enabling this capability prevents the Conditionals and Loops interface in Composer Pro programming from being displayed for a single bulb driver.  While rarely used, this property is useful when configuring smart bulb hubs where only one light serves as a controllable/configurable hub entity in the project. This capability can be changed through a `DYNAMIC_CAPABILITIES_CHANGED`  notification.


### Signature

`<hide_proxy_conditionals_loops></hide_proxy_conditionals_loops>`


### Type

Boolean. 


### Dynamic Capability

Yes


### Example

```xml
<capabilities>
    <hide_proxy_conditionals_loops>true</hide_proxy_conditionals_loops>
</capabilities>
```
